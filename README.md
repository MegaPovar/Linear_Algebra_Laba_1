# Лабораторная работа 1: решение СЛАУ

## Постановка задачи

Цель работы - реализовать набор функций для решения систем линейных алгебраических уравнений методом Гаусса и методом LU-разложения, сравнить время работы алгоритмов и показать преимущество LU-разложения при решении систем с одинаковой матрицей и разными правыми частями.

Рассматривается система:

```text
A * x = b
```

где `A` - матрица коэффициентов, `b` - правая часть, `x` - неизвестный вектор.

## Реализованные алгоритмы

В проекте реализованы:

- метод Гаусса без выбора ведущего элемента;
- метод Гаусса с выбором главного элемента по столбцу;
- LU-разложение без перестановок;
- прямая подстановка для нижнетреугольной системы;
- обратная подстановка для верхнетреугольной системы;
- вычисление невязки и относительной погрешности;
- замер времени работы алгоритмов.

### Метод Гаусса без выбора ведущего элемента

Метод выполняет прямой ход, зануляя элементы ниже главной диагонали. После этого получается верхнетреугольная система, которая решается обратной подстановкой.

В этом варианте ведущим элементом считается текущий диагональный элемент `A[k][k]`. Метод простой, но может быть неустойчивым, если ведущий элемент близок к нулю.

### Метод Гаусса с частичным выбором

Перед каждым шагом прямого хода в текущем столбце выбирается максимальный по модулю элемент. Строка с этим элементом меняется местами с текущей строкой.

Такой выбор уменьшает риск деления на очень маленькое число и обычно повышает численную устойчивость метода.

### LU-разложение

LU-разложение представляет матрицу в виде:

```text
A = L * U
```

где `L` - нижняя треугольная матрица, а `U` - верхняя треугольная матрица.

После разложения система решается в два этапа:

```text
L * y = b
U * x = y
```

Главное преимущество LU-разложения проявляется при нескольких правых частях: матрица `A` раскладывается один раз, после чего для каждого нового `b` выполняются только две быстрые треугольные подстановки.

| Файл | Назначение |
|---|---|
| `main.cpp` | Точка входа. Запускает все эксперименты. |

### 3. Проверка точности на матрицах Гильберта

Матрица Гильберта задаётся формулой:

```text
H[i][j] = 1 / (i + j - 1)
```

В программе индексация начинается с нуля, поэтому используется выражение:

```cpp
H[i][j] = 1.0 / (i + j + 1.0);
```

Точное решение задаётся как:

```text
x = (1, 1, ..., 1)^T
```

Правая часть вычисляется как `b = H * x`.

| 5 | Gauss pivot | 4.621983e-13 | 1.110223e-16 |
| 5 | LU | 5.683054e-13 | 0.000000e+00 |
| 10 | Gauss no pivot | 1.960091e-04 | 5.768888e-16 |
| 10 | Gauss pivot | 1.931427e-04 | 7.280219e-16 |
| 10 | LU | 2.866623e-04 | 5.978734e-16 |
| 15 | Gauss no pivot | 5.673623e+00 | 5.661049e-16 |
| 15 | Gauss pivot | 7.126681e+00 | 9.805224e-16 |
| 15 | LU | 1.526906e+01 | 1.451805e-15 |

На матрицах Гильберта видно, что при росте `n` относительная ошибка резко увеличивается. Это связано с плохой обусловленностью матрицы. При этом невязка остаётся маленькой, поэтому малая невязка не всегда означает малую ошибку найденного решения.

## Выводы

Метод Гаусса хорошо подходит для решения одной системы, так как не требует предварительного разложения матрицы.

Выбор главного элемента по столбцу повышает устойчивость метода, потому что уменьшает вероятность деления на маленький ведущий элемент.

LU-разложение особенно эффективно при решении нескольких систем с одной и той же матрицей и разными правыми частями. В эксперименте при `k = 100` метод Гаусса с выбором занял около `955` мс, а LU-подход - около `63` мс.

Проверка на матрицах Гильберта показывает влияние плохой обусловленности: даже при маленькой невязке фактическая ошибка решения может быть большой.
